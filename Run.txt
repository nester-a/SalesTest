1. To launch, select the web api project as an autorun.
2. Click the start button
3. The controller has two methods: get information and make a sale.
4. With the receipt of information, everything is clear.

5. The method of making a sale takes a SaleModel object, the necessary goods are stored in it in the dictionary, where the Key is Id of the product, and Value is its quantity.
It looks like this:
{
    "salesPointId": 0,           <---- id of the point of sale
    "buyerId": 0,                <---- buyer ID
    "productsToBuy": {           <---- dictionary of purchased goods
        "additionalProp1": 0,    <---- dictionary elements
        "additionalProp2": 0,
        "additionalProp3": 0
    }
}

6. In the swager, the Id is passed to the dictionary in quotation marks (example "1":1 - this means that a product with id 1 in the amount of 1 piece is required), if there are several products, they are passed by comma.
7. No comma is placed after the last dictionary element
8. If the buyer is not in the database, he is omitted in the methods.
It looks like this:
{
    "salesPointId": 0,          <---- id of the point of sale
    "productsToBuy": {          <---- dictionary of purchased products
        "additionalProp1": 0,   <---- dictionary elements
        "additionalProp2": 0,
        "additionalProp3": 0
       }
}

9. Enjoy your use






1. Для запуска выбираем проект веб-апи в качестве автозапускаемого.
2. Нажимаем кнопку запуск
3. У контроллера два метода: получить информацию и совершить продажу. 
4. С получением инфы всё понятно.

5. Метод совершения продажи принимает объект SaleModel, необходимые товары хранятся в нём в словаре, где Key Id продукта, а Value - его количество.
Выглядит он так:
{
  "salesPointId": 0,            <---- идентификатор точки продажи
  "buyerId": 0,                 <---- идентификатор покупателя
  "productsToBuy": {            <---- словарь покупаемых товаров
    "additionalProp1": 0,       <---- элементы словаря
    "additionalProp2": 0,
    "additionalProp3": 0
  }
}

6. В свагере в словарь Id передаётся в ковычках (пример "1":1 - это значит что требуется товар с id 1 в количестве 1 штука), в случае если товаров несколько они передаются черед запятую.
7. После последнего элемента словаря запятая НЕ СТАВИТСЯ
8. В случае если покупателя нет в базе данных, он в методы опускается.
Выглядит это так:
{
  "salesPointId": 0,            <---- идентификатор точки продажи
  "productsToBuy": {            <---- словарь покупаемых товаров
    "additionalProp1": 0,       <---- элементы словаря
    "additionalProp2": 0,
    "additionalProp3": 0
  }
}
